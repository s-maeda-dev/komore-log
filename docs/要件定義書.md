# komore-log（こもれろぐ） 要件定義書

## 1. 概要

**komore-log（こもれろぐ）** は、SNSのように気軽に「日々の感情」や「短い思い」を投稿できる“つぶやき日記アプリ”である。
「木漏れ日（こもれび）」のように、忙しい日々の中で感じた小さな光や影をそのまま残せる場所を目指している。
投稿内容はOpenAI APIにより解析され、アプリが共感や労いのメッセージを自動生成する。
ユーザーは、自身の投稿とAIからの返信を時系列（新しい順）で閲覧できる。

***

## 2. 目的

- 誰でも無理なく感情ログを残せるシンプルな記録体験を提供する。
- AIによる優しいメッセージで、利用者の心を少し軽くする。
- 他者との比較や承認欲求から離れ、自分のためのタイムラインを持てるようにする。

***

## 3. 対象ユーザー

- 長文日記を書く習慣が続かない人
- 感情整理やストレス発散のために手軽に記録したい人
- SNS疲れしているが、誰かに共感してもらいたい人

***

## 4. 機能要件

### 4.1 ユーザー機能

- 投稿機能（タイトル不要・短文入力、最大300文字）
- タイムライン（AI返信付き、最新順に表示）
- 投稿削除機能
- ログイン機能（Eメール＋パスワード、またはゲストモード）


### 4.2 システム機能

- **OpenAI API連携**
    - 投稿内容を解析し、感情に寄り添うAIメッセージを生成
    - 生成された返信は投稿データと紐づけて保存
- **タイムライン機能**
    - 投稿とAI返信を1セットでカード風に表示
    - 新しい順（降順）で一覧表示
- **バックエンドAPI**
    - 投稿作成（POST）
    - 投稿取得（GET）
    - 投稿削除（DELETE）

***

## 5. 非機能要件

### 5.1 パフォーマンス

- タイムライン取得の応答速度：1秒以内。
- 投稿数1万件以上でも安定稼働する設計とする。


### 5.2 セキュリティ

- JWTトークン認証によるアクセス制御。
- HTTPS通信の強制。
- OpenAI APIキーはサーバーサイドで安全に管理。


### 5.3 保守性

- Spring Bootを利用し、モジュール構造を明確化。
- AI連携部は外部モジュール化し、GPTモデル変更にも対応可能。

***

## 6. システム構成概要

```
[Frontend] - React / Vue (想定)
       |
[Backend] - Spring Boot (Java 17)
       |
[OpenAI API] - GPTモデル (text/chat形式)
       |
[Database] - MySQL
```


***

## 7. データ項目定義（簡易）

| テーブル名 | カラム名 | 型 | 概要 |
| :-- | :-- | :-- | :-- |
| posts | id | BIGINT (PK) | 投稿ID |
| posts | user_id | VARCHAR(64) | 投稿者識別子 |
| posts | content | TEXT | 投稿内容 |
| posts | ai_reply | TEXT | AI生成の返信内容 |
| posts | created_at | TIMESTAMP | 投稿日時（自動生成） |


***

## 8. API設計（概要）

| エンドポイント | メソッド | 説明 | リクエスト例 / 備考 |
| :-- | :-- | :-- | :-- |
| `/api/posts` | POST | 新規投稿を登録 | JSON: `{ "content": "今日は少し憂鬱です" }` |
| `/api/posts` | GET | 投稿一覧を取得（降順） | 投稿＋AI返信のリストを返却 |
| `/api/posts/{id}` | DELETE | 投稿削除 | 指定idの投稿を削除 |


***

## 9. 使用技術

- **アプリ名**: komore-log（こもれろぐ）
- **サーバーサイド**: Spring Boot 3.x
- **DB**: MySQL
- **AI連携**: OpenAI GPT API
- **認証**: Spring Security + JWT
- **ビルド管理**: Gradle または Maven
- **デプロイ環境**: AWS / Render / Vercel（任意）
