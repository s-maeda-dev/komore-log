# 📋 要件定義書：komore-log

## 1. プロジェクト概要
SNSのような気軽さで、日々の感情をタイムライン形式で記録するつぶやき日記アプリ。

## 2. MVP機能の詳細
### 2.1 投稿・閲覧機能
- **つぶやき投稿**:
    - 入力項目：内容（テキストのみ）
    - 仕様：タイトル入力は不要。SNSのように1行〜数行の短文を想定。
- **AI返信生成**:
    - 投稿後、OpenAI APIが内容を解析し、共感や労いのメッセージを生成して保存する。
- **タイムライン表示**:
    - 投稿した内容とAIの返信をセットにして、時系列（降順）で表示する。

### 3.2 UI/UX方針（Stitch活用）
- **UIスタイル**: シンプルなSNS風タイムライン。
- **使用ツール**: Stitch (UI生成AI) + Tailwind CSS。
- **操作性**: 入力フォームを一番上に配置し、投稿したら即座にタイムラインが更新されるUX。

## 4. 非機能要件
- **レスポンシブ**: PCブラウザで快適に動作すること。
- **開発効率**: Lombokを活用してコードを簡潔に保つ。
- **即時反映**: Spring Boot DevToolsを使用し、開発中の修正を即座に確認できること。

## 5. 使用技術（スタック）
- **言語**: Java 17+
- **フレームワーク**: Spring Boot 3.x
- **データベース**: MySQL (本番/開発用), H2 (テスト用)
- **画面作成**: Thymeleaf
- **ビルドツール**: Maven
