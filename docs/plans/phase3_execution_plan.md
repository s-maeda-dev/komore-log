# 📋 Phase 3 実行計画書：AI連携の実装（Gemini API）

このドキュメントは、`komore-log` の最大の特徴である「AIによる共感返信」を **Google Gemini API** を使って実現する手順をまとめたものです。

---

## 1. 概要
ユーザーのつぶやきを Gemini API に送り、優しく寄り添う返信をサーバーから受け取ってデータベースに保存するようにします。
Google の最新 AI を活用し、高い共感性と応答速度を両立した体験を構築します。

---

## 2. 詳細手順

### 3.1 環境設定
API を使うための準備とライブラリの導入を行います。

- [x] **Gemini API キーの準備**
    - Google AI Studio 等で取得したキーを用意し、環境変数 `GEMINI_API_KEY` に設定します。 (完了)
- [x] **接続設定 (`application.properties`)**
    - 環境変数からキーを読み込む設定の記述。 (完了)

### 3.2 連携サービスの実装
AI API と通信するためのコンポーネントを作成します。

- [x] **Gemini 連携用コンポーネントの作成 (`GeminiService.java`)**
    - RestTemplate を使用した API リクエスト処理の実装。 (完了)
- [x] **エラーハンドリングの実装**
    - API 呼び出し失敗時のフォールバック処理。 (完了)

### 3.3 サービス層の統合
AI 連携のロジックを既存の業務ロジックに組み込みます。

- [x] **`PostService` の拡張**
    - `savePost` とは別に `generateAiReply` を作成し、非同期的な更新を可能にする。 (完了)
- [x] **コントローラーのエンドポイント追加**
    - AI 返信生成専用の POST エンドポイント作成。 (完了)

### 3.4 プロンプトの設計
Gemini に「どのようなキャラクターで返信してほしいか」を指示します。

- [x] **システム命令（System Instruction）の設定**
    - 「癒し系のカウンセラー」としての定義、および「100文字以内」「一文ごとに改行」といった出力形式の指示。 (完了)

- [x] 日本語での自然なやり取りができている。
