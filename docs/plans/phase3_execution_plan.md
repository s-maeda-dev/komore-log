# 📋 Phase 3 実行計画書：AI連携の実装（OpenAI API）

このドキュメントは、`komore-log` の最大の特徴である「AIによる共感返信」を本物の OpenAI API を使って実現する手順をまとめたものです。

---

## 1. 概要
ユーザーのつぶやきを OpenAI のサーバーに送り、優しく寄り添う返信をサーバーから受け取ってデータベースに保存するようにします。

---

## 2. 詳細手順

### 3.1 OpenAI API の準備と環境設定
API を使うための鍵（API Key）を設定し、アプリから安全に呼び出せるようにします。

- [ ] **OpenAI API キーの取得と確認**
- [ ] **環境変数（または application.properties）への設定**
    - **初心者解説**: API キーは「銀行の暗証番号」のような大切なものです。プログラムに直接書かず、安全に隠して管理する方法を学びます。

### 3.2 OpenAI 連携用クライアントの作成
OpenAI のサーバーと通信するための専用のクラスを作ります。

- [ ] **`OpenAiClient.java` の作成**
    - パッケージ: `com.komore.komorelog.client`
    - 機能: 投稿テキストを送り、生成された返信テキストを受け取る。
    - **使用する技術**: Spring Boot の `RestTemplate` または `WebClient`（HTTP 通信用）。

### 3.3 PostService への組み込み（仕上げ）
Phase 2 で作った「仮の返信」を、本物の AI の返信に置き換えます。

- [ ] **`PostService.java` の修正**
    - 保存処理（`savePost`）の中で、`OpenAiClient` を呼び出すように変更する。
    - **初心者解説**: これで、投稿ボタンが押された瞬間に「AIに相談して返事をもらい、それをセットにして保存する」という一連の流れが完成します！

---

## 3. 完了条件
- [ ] アプリを起動し、API を通じて投稿した際、AI から生成された返信が届いている。
- [ ] データベース（MySQL）に、AI からの返信が正しく保存されている。

---
