# 🚀 komore-log 開発実行計画書（MVPフェーズ）

本計画書は、`docs/要件定義書MVP.md`に基づき、アプリの完成までの手順をフェーズごとに区切ったものです。初心者の方でも一歩ずつ進められるよう、設計から実装、動作確認までの流れを整理しました。

---

## 📅 開発フェーズ一覧

### Phase 1: データベース・基盤の準備
**目標**: データを保存する場所と、Javaから接続するための設定を完了する。

1.  **MySQL データベースの作成**
    *   `komore_db` という名前のデータベースを新規作成する。
2.  **Spring Boot の設定更新 (`application.properties`)**
    *   MySQLへの接続情報（URL, ユーザー名, パスワード）を記述する。
3.  **エンティティ（モデル）の作成**
    *   `Post` クラスを作成：投稿ID、内容、AI返信、日時を定義する。
    *   **初心者解説**: Javaのクラスとデータベースのテーブルを紐付ける作業です。

### Phase 2: メイン機能の実装（バックエンド編）
**目標**: 投稿を保存・取得・削除する仕組み（API）を作る。

1.  **リポジトリの作成**
    *   データベースを操作するためのインターフェースを作成する。
2.  **サービスの作成**
    *   ビジネスロジック（データの保存や削除のルール）を記述する。
3.  **コントローラーの作成**
    *   ブラウザや外部からのリクエストを受け取る窓口（API）を作成する。
    *   この時点では、AI返信は「固定のサンプル文字」を返すように仮実装する。

### Phase 3: AI連携の実装（OpenAI API）
**目標**: 投稿内容に合わせてAIが返信を生成するようにする。

1.  **OpenAI API キーの準備**
    *   環境変数にAPIキーを設定する。
2.  **AI連携用クライアントの作成**
    *   OpenAIのAPIを呼び出すためのプログラムを作成する。
3.  **AI返信ロジックの統合**
    *   投稿保存時に、AIに問いかけて返答を `ai_reply` カラムに保存するように修正する。

### Phase 4: 画面の実装（フロントエンド編）
**目標**: ブラウザから実際に操作できる見た目を作る。

1.  **タイムライン表示用 HTML (Thymeleaf)**
    *   投稿カードが並ぶトップページを作成する。
2.  **投稿用フォームの作成**
    *   テキストを入力して送信できるフォームを作成する。
3.  **UIのブラッシュアップ (Tailwind CSS)**
    *   「木漏れ日の静けさ」を感じさせる落ち着いたデザインを適用する。

### Phase 5: 総仕上げと動作確認
**目標**: すべての機能が連携して動くことを確認する。

1.  **統合テスト**
    *   投稿して、AIの返信が返ってきて、削除ができるか一連の流れを確認する。
2.  **エラー処理の追加**
    *   投稿が空だった場合や、AI APIがエラーになったときの対応を追加する。
