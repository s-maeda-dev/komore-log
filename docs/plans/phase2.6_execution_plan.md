# 📋 Phase 2.6 実行計画書：簡易フロントエンド（動作確認用）の作成

このドキュメントは、バックエンドの API が正しく動作しているかをブラウザ上で確認するための、簡易的な画面（フロントエンド）を作成する手順をまとめたものです。

---

## 1. 概要
Java のテンプレートエンジン「Thymeleaf」を使用し、Java プログラムから直接 HTML 画面を表示できるようにします。
バリデーションエラーの表示や、投稿一覧のリアルタイム更新（簡易版）を確認できる環境を整えます。

---

## 2. 詳細手順

### 2.1 画面用コントローラーの作成
API ではなく、HTML 画面を返すための専用コントローラーを作成します。

*   **クラス名**: `ViewController.java`
*   **パッケージ**: `com.komore.komorelog.controller`
*   **パス**: `/` (トップページ)
*   **役割**: `index.html` というテンプレートを表示するように指示します。

### 2.2 メイン画面の作成 (`index.html`)
ユーザーが実際に触れる画面を作成します。

*   **ファイルパス**: `src/main/resources/templates/index.html`
*   **内容**:
    *   大きなタイトル「こもろぐ komore-log」
    *   投稿用の入力フォーム (テキストエリア)
    *   送信ボタン
    *   バリデーションエラーを表示するエリア
    *   投稿一覧を表示するリストエリア

### 2.3 JavaScript による API 連携
画面のボタンを押した時に、バックエンドの API (`/api/posts`) を呼び出す仕組みを作ります。

*   **機能**:
    *   送信ボタンクリック：入力内容を API に送る。
    *   エラーハンドリング：API から「400 Bad Request」が返ってきたら、そのエラーメッセージ（例：「投稿内容は必須です。」）を画面に赤字で表示する。
    *   一覧更新：保存に成功したら、画面を再読込して最新の投稿を表示する。
